# Admin Clearout Template

## Suggested Flow

1. **Understand** — What are we clearing and what's the scope?
   - Inbox? To-do list? A specific category? All of the above?
   - How much time do we have?
   - What's the priority (urgent first, or quick wins first)?

2. **Categorize** — Sort items into buckets
   - **Do now**: Takes < 2 minutes or is urgent
   - **Schedule**: Important but not urgent — assign a time
   - **Delegate**: Someone else should handle this
   - **Defer**: Need more info or can wait
   - **Delete**: No longer relevant

3. **Execute** — Process each bucket in order
   - Handle "Do now" items immediately
   - Schedule "Schedule" items with specific times
   - Delegate items with clear instructions
   - Note what "Defer" items are waiting on
   - Delete without guilt

4. **Review** — Check that everything is processed
   - Any items that fell through the cracks?
   - Did we address all urgent items?
   - Is there a clear plan for deferred items?
   - Should we set a recurring clearout?

## Admin-Specific Success Criteria

- [ ] Inbox/list is at zero (or close to it)
- [ ] Every item has a next action or is removed
- [ ] Urgent items are handled, not just acknowledged
- [ ] Deferred items have a specific "when" attached

## Tips

- The 2-minute rule: if it takes less than 2 minutes, do it now
- Don't re-read items you're not ready to process — that's procrastination
- Delete aggressively — most saved-for-later items are never needed
- For email: archive, don't file into folders. Search is faster than sorting.
- If there are more than 50 items, do a quick triage pass first (urgent/not urgent), then process
- recurring clearout: If this pile builds up regularly, schedule a weekly clearout
